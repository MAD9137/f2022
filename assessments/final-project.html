# Description

The final project will use the GitHub search api to return results of a user search request submitted through the application and display the results in a list. Selecting a list item will navigate the user to a detail page that presents additional information about each user. The application will also include an about view that presents information about you and and a settings view where users can configure persistent settings.

The design of the application is yours to make but keep in mind you must work within the requirements described below.

# Approach

Main components of this app will be built/modified during class and students will complete required components as instructed. In broad terms, we'll build out the project structure, navigation and network data fetches together in class and touch on new topics not covered. Students should expect that elements of this project will require them to review source documentation and attempt to devise solutions using trial and error.

# SwiftUI Views

The application must include the following SwiftUI views:

  1. A list item view (row view) that contains basic user information including:

    • avatar
    • username

  2. A list of users returned in the search

    • the number of items will be constrained by the "results per page" setting
    • list items are instances of the list item view

  3. A detail view for each user that includes:

    • a picture of the user's avatar
    • user's name, location and companies
    • follower, gist and repo counts
    • last updated and account created dates

  4. An about view that presents information about you including:

    • an image
    • App name and version/build id
    • "Created by: "
    • Company name
    • Email address
    • Website url

  5. A settings view for configuring the following persistent search settings:

    • minimum number of repos (Stepper)
    • minimum number of followers (Stepper)
    • number of results per page (Stepper)

  6. ContentView as the entry point of the app and search data entry.

# Data Modelsnew

  1. A User struct in a separate file that serves as a data model for the user details view
  2. A UserSearch struct that serves as a data model for user search results
  3. A UserResponseSearch struct that serves as a data model for the entire user search response object

NOTE

We will not be adding MVVM view-models to our SwiftUI views for the final. All functionality will be contained in Views and/or extensions.

# Project Structure

  1. Create a new XCode iOS App Project using Swift and SwiftUI

  2. Add the following packages using the Xcode Swift package manager:

  • https://github.com/Alamofire/Alamofire
  • https://github.com/onevcat/Kingfisher
  1. Add the following SwiftUI View files to a new group named "Views":
  • AboutView
  • UserSearchResultsView
  • UserDetailsView
  • AppSettingsView
  • ListItemView
  1. Add the following new Swift files to a new group named "Models":
  • UserModel.swift

# Improving the UI/User Experience:

The UI for the ContentView (start page) is very basic.

  1. Redesign the UI so that it not only looks better but also provides a better user experience.

Feel free to add/remove View controls but make sure your design works well in both portrait and landscape orientations.

Add the following items to improve the user experience:

  1. ProgressView in ContentView - visible during the user search and hidden when completed

Visible message informing the user that no results were found for a user search - this message should disappear after the user changes the search text or after a short length of time.

  1. Use @AppStorage property wrappers to save and load the following 3 required values in ContentView:
  • minimum number of repos
  • minimum number of followers
  • number of results per page

# Code quality, comments and MARK annotations:

  • Source code should be professional and correctly formatted.
  • Add your name and the date at the top of all Swift source code files in this project.
  • Add separate MARK annotations in all Swift source code files in the project the properties and method regions.
  • Also add MARK annotations for other significant areas of code.
  • Add appropriate comments to your code for new technologies covered in this project. These comments should be written so that you can easily understand how to re-use it in another project.
  • Make sure you check each Swift source file for issues and fix any warnings etc. that can be made without changing functionality.
  • Remove ALL code you have commented out or are not using in the project.
  • Remove ALL TODO annotations.
  • Remove ALL unused variables, methods and unnecessary nil checks from your source code.
  • Remove ALL unused imports from your source code.
  • Remove ALL unreachable code from your source code.

# Considerations

  1. Feature creep must be nil unless requirements are fulfilled
  2. Any and all code is subject to assessment
  3. No matter how descriptive/prescriptive requirements are, there is always room for interpretation. When there is doubt or requirements are unclear, ask.
  4. Time permitting, we may explore localization and include it in the project.
