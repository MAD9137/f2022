# Description

In this assignment you will create an iOS App using Swift/SwiftUI that will first check to see if the device it is running on is connected to the internet. If it is not connected it will display an error message. If it is connected it will display a random image download from the internet.

The user can then use a Left to Right Drag gesture to download another random web image.

There will be a minimum of 20 image URL's stored in a list. These URL's are the source addresses for the images used in this App.

You must find, test and then add these URL's to your App.

NOTE

All URL's must use the secure https:// protocol. For example: https://source.unsplash.com/random

Also add the following "dead" URL to you list for testing. This URL must be active (not commented out) in your URL list for your submitted assignment so that error handling can be verified:

https://placezombie.com/720x405

You will create a class that contains a method that will deliver a random URL from your list which will then be used as the source for the image that is to be displayed.

This method will provide all twenty URL's (plus the dead URL) in the list separately, randomly and without duplicating a URL.

Once all URL's have been used, the list will shuffle then repeat this process.

This app will use the Kingfisher Library to display images from the web.

# Requirements

# Dependencies

Using the Swift Package Manager in Xcode add the latest version of the following libraries from GitHub:

  • Kingfisher library (opens new window)
  • the ToastSwiftUI package (opens new window)

# User Interface

The following controls will be added to the ContentView.swift file (the user interface):

  • KFImage
  • ProgressView
  • ZStack
  • VStack
  • Color

# Separate Files

Use separate files for the ImageUrls class and InternetConnection class.

# Properties

You will add several properties to the ContentView.swift file including:

  • Several @State property wrappers
  • a private ImageUrls object named imageUrls that will be used to retrieve image URLs (all URLs must be https)
  • a private InternetConnection object named internetConnected that will be used to check for internet access

# Enum

An enum of type String called toastMessages will be added and will contain values and raw values for the following cases:

  • clearedCache
  • imageProcessingOn
  • imageProcessingOff
  • loadFailed

The raw values from the enum described above will be used as message text displaying the required user information messages (toast style).

# Gestures

Add the following required gestures to the ZStack:

  • DragGesture - used to load images using a Left to Right drag gesture
  • LongPressGesture - used to empty the image cache
  • onTapGesture - set as a double tap - used to apply image processing

# Additional Required Features

  • Alert message
  • Image cache deleted on startup
  • Some sort of UI messaging similar to Toast messages used in Android Listener for image downloads. You may build your own solution or use the ToastSwiftUI library.
  • Use SwiftUI image modifiers to fill the safe viewing area of the device.
  • At least one image processor
  • Background color set to Black for entire viewing area including outside the safe viewing area
  • ProgressView color change and optional size change
